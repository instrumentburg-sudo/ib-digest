OpenAI вынес Codex и frontier-модели в AWS

Что случилось OpenAI открыл доступ к Codex и своим frontier-моделям через Amazon Bedrock. Сигнал не в интеграции как таковой, а в том, что агентную разработку теперь проще покупать, согласовывать и выкатывать внутри стандартного корпоративного контура.

Живой пример https://openai.com/index/openai-frontier-models-and-codex-are-now-available-on-aws/

👉 Как применить: держать это как аргумент для B2B-подачи и будущих корпоративных интеграций: «тот же агентный контур, но через привычный AWS-процесс закупки и безопасности».

Stanford выложил жесткие правила для ИИ-агентов в обучении

Что случилось в CLAUDE.md для stanford-cs336/assignment1-basics прямо прописали, что агент может объяснять, направлять и помогать с отладкой, но не писать код, не закрывать TODO и не запускать команды за студента.

Живой пример https://github.com/stanford-cs336/assignment1-basics/blob/main/CLAUDE.md

👉 Как применить: это готовый шаблон правил для наших собственных агентных контуров, где нужно отделить «помогает думать» от «делает работу вместо человека».

В трендах GitHub выстрелил Compound Engineering plugin

Что случилось официальный плагин Every для Claude Code, Codex и Cursor вышел в ежедневные тренды GitHub с 417 звездами за день. Акцент у них не на одном агенте, а на цикле: продумать, спланировать, сделать, проверить, зафиксировать знание.

Живой пример https://github.com/EveryInc/compound-engineering-plugin

👉 Как применить: забрать себе сам паттерн цикла. Для OpenClaw это хороший ориентир, как упаковывать повторяемую инженерную работу в навыки и ревью, а не только в промпты.

В трендах GitHub поднялся Harness для сборки команд агентов под домен

Что случилось revfactory/harness набрал 524 звезды за день. Репозиторий генерирует не одного агента, а целую команду под задачу: роли, паттерн координации и навыки для каждого участника.

Живой пример https://github.com/revfactory/harness

👉 Как применить: полезно как референс для наших мульти-агентных схем. Особенно там, где нужен не просто вызов сабагентов, а повторяемая архитектура команды под конкретный процесс.
