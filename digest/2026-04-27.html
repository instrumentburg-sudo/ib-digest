CUA — открытая инфраструктура для компьютерных агентов

Что случилось в GitHub Trending резко выстрелил trycua/cua — открытый стек для агентов, которые управляют полноценным рабочим столом, с песочницами, SDK и бенчмарками.

Живой пример https://github.com/trycua/cua

👉 Как применить: подсмотреть их подход к изоляции и воспроизводимым средам для наших браузерных и desktop-агентов.

Beads — память для кодинг-агента

Что случилось в трендах всплыл beads — проект, который делает долговременную память и контекстные связи для кодового агента, чтобы он меньше терял нить между задачами.

Живой пример https://github.com/gastownhall/beads

👉 Как применить: полезно для развития нашей typed-memory схемы и длинных многошаговых задач, где агент забывает старые решения.

EvanFlow — TDD-цикл поверх Claude Code

Что случилось на Hacker News обсуждают EvanFlow — проект, который заворачивает Claude Code в жёсткий цикл «тест → правка → повтор», чтобы меньше гадать и быстрее ловить регрессии.

Живой пример https://github.com/evanklem/evanflow

👉 Как применить: взять как паттерн для наших кодовых агентов, где нужен минимальный безопасный gate перед финальным ответом.

SWE-bench перестал быть нормальной меркой фронтира

Что случилось OpenAI публично написал, что SWE-bench Verified больше не измеряет реальные возможности передовых кодовых моделей; на HN это активно разбирают как сигнал, что старые метрики начинают врать.

Живой пример https://openai.com/index/why-we-no-longer-evaluate-swe-bench-verified/

👉 Как применить: меньше смотреть на красивые бенчмарки в лоб и больше проверять агентов на наших реальных задачах: браузер, интеграции, долгие контексты, cron и восстановление после сбоев.
